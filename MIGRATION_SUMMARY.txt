API ENDPOINTS - BEFORE & AFTER
================================================================================

BEFORE (Mock System)
─────────────────────────────────────────────────────────────────────────────

GET /api/branches
  Response: mockBranches array (hardcoded)
  Speed: Instant
  Data: Always same 5 branches
  Real-time: No

POST /api/login
  Process: mockUsers lookup + manual session creation
  Session: Stored in Map<sessionId, session>
  Security: Basic (no password hashing)
  Real-time: No

POST /api/upload
  Process: Simulates file processing
  Storage: Nowhere (lost on server restart)
  Errors: Simulated random errors
  Real-time: No


AFTER (Supabase System)
─────────────────────────────────────────────────────────────────────────────

GET /api/branches
  Query: SELECT * FROM branches WHERE is_active = true
  Speed: Fast (indexed)
  Data: From database (customizable)
  Real-time: Yes (via Supabase Realtime if configured)
  Security: RLS policy enforcement

POST /api/login
  Process: supabase.auth.signInWithPassword()
  Session: Supabase Auth (secure, httpOnly)
  Security: Enterprise-grade (bcrypt, JWT)
  Real-time: Yes (auth state changes)

POST /api/upload
  Process: Save to database
  Storage: PostgreSQL (persistent)
  Errors: Database records (queryable)
  Real-time: Yes (via Supabase if configured)
  Backup: Supabase handles

BEFORE (Mock System)
───────────────────────────────────────────────────────────────────────────────

  ✗ Passwords stored in code (mockUsers)
  ✗ No password hashing
  ✗ Sessions in memory (not secure)
  ✗ No token validation

Data Access:

  ✗ Any user can see all data

  ✓ JWT token validation
  ✓ Token refresh mechanism

  ✓ Row Level Security (RLS) policies
