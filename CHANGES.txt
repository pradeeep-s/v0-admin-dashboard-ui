ARCHITECTURE
================================================================================

Authentication Flow:
  1. User visits /login
  2. Submits credentials via POST /api/login
  3. Server validates and creates session cookie
  4. Browser stores httpOnly cookie automatically
  5. Client redirects to /dashboard
  6. Dashboard layout checks /api/me for auth
  7. If 401, redirect back to /login
  8. If 200, load user data and render UI
  9. All API calls include credentials (cookies)
  10. On logout, clear cookie and redirect to /login

Role-Based UI:
  1. useAuth hook fetches /api/me
  2. Gets user with role (Admin/Operator)
  3. Sidebar filters menu items by role
  4. Admin Configuration only shows for Admin
  5. Navbar displays username and role

API Integration:
  1. All dropdowns fetch from API
  2. No hardcoded data in components
  3. Loading states during fetch
  4. Error handling with toast notifications
  5. Real-time data sync between selections
