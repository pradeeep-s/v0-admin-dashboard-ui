AUTHENTICATION FLOW
================================================================================

BEFORE (Hybrid - Removed):
  1. Try Supabase Auth
  2. If fail, try users table
  3. If fail, create user

AFTER (Simple - Current):
  1. Query users table by email
  2. Verify password with bcrypt
  3. If not found, create user
  4. Return user with role

Result: Single, simple flow. No Supabase Auth dependency.

================================================================================
DATABASE (users table)
================================================================================

Schema:
  id              UUID PRIMARY KEY
  email           TEXT UNIQUE (used for login)
  username        TEXT UNIQUE
  password_hash   TEXT (bcrypt encrypted, NOT plain text)
  role            TEXT ('Admin' or 'Operator')
  is_active       BOOLEAN (true = can login, false = blocked)
  last_login      TIMESTAMP
  created_at      TIMESTAMP

Demo Data (after setup):
  admin@pacs.com / admin123 (Role: Admin)
  operator@pacs.com / operator123 (Role: Operator)

No Supabase Auth tables needed - users table is the single source of truth!

API ENDPOINTS
================================================================================

POST /api/login
  Input: { email: "...", password: "..." }
  Output: { success: true, data: { id, username, email, role } }
  Auth Method: Users table bcrypt verification
  Response: 200 (success), 401 (invalid credentials), 403 (inactive)

POST /api/register
  Input: { email: "...", password: "...", role?: "..." }
  Output: { success: true, data: { id, username, email, role } }
  Auto-creates user if not exists
  Default role: Operator
  Response: 200 (success), 400 (invalid), 409 (exists)

GET /api/me
  Output: { success: true, data: { id, username, email, role } }
  Requires: Valid session
  Response: 200 (success), 401 (unauthorized), 404 (not found)

POST /api/logout
  Output: { success: true, message: "Logout successful" }
  Response: 200 (success)
